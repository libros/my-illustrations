# QA Checklist

检查时使用**当前变体**的 IP 显示名与风格要求（默认：小黑 + 纯白手绘）。

## 必过项

- 是 16:9 横版。
- 背景与线稿符合当前风格文件（默认：干净白底 + 黑色手绘线）。
- 出现了正确的当前 IP 角色（外形符合该 IP 文件，不要画成别的 IP）。
- 角色承担核心动作，不只是装饰。
- 性格仍是认真、deadpan、不卖萌（见 `personality.md`）。
- 没有复刻旧案例构图，而是为当前文章生成了新隐喻。
- 画面怪诞、有创意、有意思。
- 简洁清爽，主体不超过画面约 60%。
- 一张图只讲一个核心结构。
- 标注少、短、能读，且**语言符合锁定的 label_lang**（zh / en / bilingual）。
- 颜色符合当前风格（默认：橙=主路径，红=重点/问题，蓝=补充）。

## 失败信号

出现以下情况，重生成或局部编辑：

- 左上角有“常见坑 / Workflow / 系统架构图 / 路线图”等标题。
- 角色像吉祥物、表情包或可爱卡通。
- 画成了错误 IP（例如要便签却画成小黑）。
- 画面像 PPT、课程课件、正式流程图。
- 元素太多、箭头太多、节点太多。
- 文字变成大段解释。
- 背景/质感违反当前风格文件。
- 真实 UI 截图或科技感界面。
- 标注语言错了（例如锁定英文却满屏中文，或反之）。
- 文字错字严重或标注不可读。
- 画面太死板，没有荒诞隐喻。
- 和 `assets/examples/` 里的旧案例构图过于相似。

## 迭代方法

- 太普通：让当前角色成为动作主体，加入一个奇怪但成立的隐喻。
- 太复杂：删节点，只保留一个动作和 3-5 个短标注。
- 太可爱：强调 deadpan、blank serious expression、not cute、not mascot；重读 `personality.md` 与当前 IP 专属禁止。
- 太 PPT：去掉标题、边框、整齐网格和过多箭头，改成手绘场景。
- 太像旧案例：保留核心意思，换掉主物件和角色动作。
- IP 漂移：重贴当前 IP 的英文 prompt 块再生成。
- 文字错：优先局部编辑；错得多就重生成并减少标注数量。

## 交付判断

高质量图应该让读者先觉得“有点怪”，然后 1 秒内看懂结构。

如果第一眼像教程页，而不是当前风格下的怪诞产品草图，就不合格。
